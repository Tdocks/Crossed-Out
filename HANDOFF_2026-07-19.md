## 6. Conventions for the next engineer / agent

- Repo path: `/Users/tylerdockswell/Projects/Crossed Out ` — **the
  trailing space is real**, always quote it.
- New Swift files require `cd "/Users/tylerdockswell/Projects/Crossed Out " && xcodegen generate`
  before building (`project.yml` globs the `CrossedOut/` folder). Editing
  existing files does not.
- Build to verify:
  ```
  xcodebuild -project CrossedOut.xcodeproj -scheme CrossedOut \
    -destination "generic/platform=iOS Simulator" build
  ```
- DB changes are always a **new numbered migration**, applied by a human
  via `psql` — never apply migrations from an agent session directly.
- RLS / role model follows `0021_rbac_and_church_portal.sql`:
  `system_admin` / `church_admin` / `user` roles; `is_system_admin()` and
  `current_account_status()` helpers; own-rows RLS
  (`auth.uid() = user_id`) for user-owned tables; privileged writes go
  through `SECURITY DEFINER` RPCs, never direct table grants; no
  self-escalation paths.
- Design system tokens live in `CrossedOut/DesignSystem/` (`coPaper`,
  `coCard`, `coInk*`, `coCrossRed`, `coOlive`, `coGold`, `coDivider`,
  `.coDisplay`/`.coScripture`/`.coUI` fonts, `COCard`/`COPrimaryButton`/
  `COSecondaryButton`/`COIcon`/`CODivider` components) — reuse them,
  never hard-code colors or fonts.
- Deterministic-first, always: AI is gated behind explicit user action and
  capped server-side (see `kyra_usage`, `devotional_ai_usage` patterns).
  New features should default to rule-based logic and only reach for AI
  when nothing deterministic can do the job.
